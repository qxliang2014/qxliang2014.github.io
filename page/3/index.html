搭建托管在Github上的Hexo博客

发表于 2018-04-07 更新于 2021-03-21 2.4k 2 分钟

¶前言

最近有搭建个人博客系统的需要，搜索翻阅网上这类教程繁多，但是未必适用于本机早已过时的Ubuntu 12.04LTS系统。下面简要记录下使用SSH，Git，Nodejs等工具搭建起不失美观的Hexo博客系统并托管于Github的实现过程，表达尽量简明，以作备忘参考。

¶环境检查与准备

¶Git

程序员利器，可以在终端用以下命令检查是否已安装:

git --version 或 git --help

如果是首次安装则可运行：

sudo apt-get install git

¶SSH

SSH是Ubuntu标配了，把主机作为SSH协议的本地客户端，而github作为远程服务器。因为采用基于密钥的安全认证，所以先执行以下命令创建一对密钥，分别位于～/.ssh/id_rsa，~/.ssh/id_rsa.pub，然后把其中的公有密钥放到github上自己的宿主目录中（详见下方）
ssh-keygen -t rsa -b 4096 -C "Github的注册邮箱地址"
如果已经生成过密钥可按提示选择overwrite覆盖，另外需要添加以下内容到～/.ssh/config文件中（如果没有这个文件就新建一个）：

Host github.com
User qxliang01@gmail.com //更改为你的github注册邮箱
Hostname ssh.github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
Port 443

¶Github

准备好Github的帐号并登录，然后在Github页面的右上方选择New repository新建一个仓库，在仓库名字输入框中输入：

想要的名字.github.io

点击Create repository创建即可。
另外，用记事本打开刚才获得的～/.ssh/id_rsa.pub，复制里面的全部内容，进入
https://github.com/settings/ssh，将复制的内容粘贴到Key的输入框,点击Add SSH key按钮即可。

¶Node.js

Hexo程序是基于Node.js实现的，鉴于Node.js的多版本问题，较好的方式是先下载安装Node.js多版本管理工具NVM，然后用NVM安装稳定版本的Node.js。
curl https://raw.githubusercontent.com/creationix/nvm/master/install.sh | sh
nvm install stable

但是依次执行这两个命令会踩到坑，可能提示没找到命令，这就需要配置并激活一下nvm的环境变量：

echo ". ~/.nvm/nvm.sh" >> ~/.bashrc

source ~./bashrc
如果出现其他相关问题，可以参考nvm的git主页 。
也可以用n工具来管理多版本Node.js，这里不再展开。

¶安装与部署Hexo

¶Hexo安装

以上依赖环境安装好后，使用Node.js自带的包管理器npm来执行Hexo的安装：

npm install -g hexo-cli

¶初始化博客系统

创建博客文件夹，生成静态网页：

hexo init 文件夹名 //比如命名为BlogA

cd BlogA

npm install //安装依赖

hexo g //生成静态网页（第一次执行会生成一个public目录）

此时用ls -l查看BlogA目录，会看到里面包含一个名为_config.yml的文件，下面将对其进行关键的修改。

¶配置和部署博客

先来配置Github，还是在博客目录下，执行：

git config --global user.name "username" //username为个人的github用户名

git config --global user.email "email@example.com"//email@example.com为个人的github注册邮箱

接着就是将博客部署到github了，用vim打开_config.yml配置文件，找到deploy项，作如下修改（注意冒号后面有一空格）：

deploy:
type: git
repo: https://github.com/username/username.github.io.git //两个username和上面的一致
branch: master

然后在命令行中执行：

hexo d

即可完成部署。

¶踩坑提醒：

有时碰到不能部署到git上的情况，那么安装以下插件或许会有帮助：

npm install hexo-deployer-git --save

还有可能是SSH连接没建立好，那么执行测试：

ssh -T -p 443 git@github.com
如果提示是否继续连接，就敲“yes”，出现：

Hi qxliang2014! You’ve successfully authenticated, but GitHub does not provide shell access

就说明成功了。否则，可能还要修改etc/ssh/ssh_config配置文件，查看端口是否为默认值，并把报错的配置项用#号注释掉。

¶🎉开始创作发布个人技术博客!

新建一篇博文：

hexo new post "mypost1"
这样就在博客文件夹的source/_posts下生成了一个mypost1.md文件，用很赞🌟的Markdown语法来编辑生产，完成后再用hexo g、hexo d推送到github。

最后就可以在浏览器上输入：username.github.io来访问个人博客了！

¶温馨提示

如果上面有个别命令执行出现错误，那么换用root权限（sudo)试试看。
